# SGLG Monitoring System

*City Planning and Development Office (CPDO)*

## End-User Manual

**Audience:** Daily users (Office users, staff reviewers, admins using app features)  
**System Type:** Web application  
**Last Updated:** May 5, 2026

---

## 1. Introduction

The **SGLG Monitoring System** (provided by CPDO) helps offices submit and track checklist compliance documents, review status, and monitor governance progress.

This guide focuses on how to use the app in day-to-day operations.

---

## 2. Accessing the System

### 2.1 Login

1. Open the system URL in a browser.
2. Enter your email and password.
3. Click **Login**.

If credentials are valid, you will be redirected to the dashboard.

### 2.2 Forgot Password

1. Click **Forgot Password** on the login page.
2. Enter your registered email.
3. Follow the reset link sent to your email.
4. Set your new password and log in again.

### 2.3 Session Timeout

- For security, inactive sessions are automatically expired.
- If you are idle too long, you will be redirected to login.
- Save your work before leaving your session unattended.

---

## 3. User Roles and Access

Your account role controls which pages and actions are available.

### 3.1 ADMIN

- Manages users, templates, governance setup, years
- Can access high-level management pages and system controls

### 3.2 STAFF

- Supports operations, submissions, reports, offices, and monitoring
- Can review and track compliance-related workflows

### 3.3 OFFICE

- Submits compliance files for assigned office scope
- Tracks own office checklist and submission status

If you cannot access a page, it is usually due to role restrictions.

---

## 4. Main Navigation and Pages

Depending on role, the system contains the following major pages:

- **Dashboard**: KPI cards, trends, and activity summary
- **Submissions**: Upload and monitor checklist submissions
- **My Checklists** (Office role): Required checklist items and progress
- **Templates / Categories / Manage** (Admin role): Template configuration
- **Governance** (Admin role): Governance area setup and compliance management
- **Users** (Admin role): User management
- **Offices** (Admin/Staff): Office information and assignments
- **Reports** (Admin/Staff): Exportable monitoring outputs
- **File Manager** (Admin/Staff): File-level handling
- **Notifications**: System updates and workflow alerts
- **Audit Logs**: Operational traceability
- **Profile / Settings**: Personal account information

---

## 5. Submission Workflow (Office Users)

### 5.1 View Required Checklist Items

1. Go to **My Checklists** or **Submissions**.
2. Select applicable year/governance filters if available.
3. Review expected checklist items and due requirements.

### 5.2 Upload a File

1. Locate the checklist item.
2. Click upload action.
3. Choose allowed file type (PDF/JPG/PNG/DOCX/XLSX).
4. Confirm upload.

### 5.3 Replace an Existing File

- Uploading a new version updates the current file for that submission.
- Previous versions are retained in history (system-level versioning).

### 5.4 Track Submission Status

Statuses include:

- **PENDING**
- **APPROVED**
- **DENIED**
- **REVISION_REQUESTED**

If a revision is requested, review notes/comments and submit an updated file.

---

## 6. Review Workflow (Staff/Admin)

### 6.1 Review Submission

1. Open submission details.
2. Validate file and checklist context.
3. Apply review action:
   - Approve
   - Deny
   - Request revision

### 6.2 Record Decision Notes

- Add clear remarks for each decision.
- Use specific feedback when requesting revision.

### 6.3 Monitor Follow-ups

- Track pending and revision-requested items in dashboard and reports.
- Check notifications for newly updated submissions.

---

## 7. Notifications

The system sends in-app notifications for key workflow events, such as:

- submission received
- approved/denied/revision requested
- system updates and related alerts

### Best Practice

- Review unread notifications daily.
- Use links in notifications to jump directly to relevant records.

---

## 8. Reports and Exports

Reports are available to authorized roles and can be exported as:

- CSV
- XLSX (for selected reports)
- PDF

Common report outputs:

- status summary
- missing uploads
- dashboard executive report
- compliance progress

If export returns no output, verify filters (year/office/governance) and whether data exists.

---

## 9. Profile and Account Practices

### 9.1 Profile Updates

- Use **Profile** page to update your personal details.

### 9.2 Password Safety

- Use strong passwords.
- Do not share credentials.
- Change password immediately if you suspect account compromise.

### 9.3 Logout

- Always log out after use, especially on shared computers.

---

## 10. Common User Issues

### Issue: Cannot log in

- Check email/password spelling
- Ensure account is active
- Use Forgot Password if needed

### Issue: Access denied on page

- Your role may not have permission
- Contact system admin for access clarification

### Issue: Upload rejected

- File may exceed size limit or have unsupported type
- Confirm format and retry

### Issue: Export fails

- Selected filters may have no matching data
- Try broader filter or different date/year scope

---

## 11. User Support Escalation

When reporting an issue to IT/Admin, include:

- your role and office
- page where problem occurred
- exact action attempted
- timestamp of incident
- screenshot (if possible)
- any visible error message

---

## 12. Quick Daily Checklist

- Log in and check notifications
- Review pending tasks/submissions
- Upload or review required files
- Confirm statuses are updated
- Export needed reports (if role permits)
- Log out securely
